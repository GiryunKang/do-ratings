## 1. FaultlineFeed (단층선 피드)

**기술:** 평점 표준편차 기반 양극화 감지 → 균열 애니메이션 + 진동 효과

**선행 기술/서비스:**
- Rotten Tomatoes의 'Splat' 아이콘, Metacritic 노란색 표시 → 정적 UI, 동적 애니메이션 없음
- 데이터 변동성 시각화 특허 다수 존재, 그러나 '평점 표준편차 + 균열 물리 메타포' 결합 사례는 드묾

**특허 가능성: 높음**
- 신규성: 데이터 분석(표준편차)과 UI 피드백(균열 효과) 결합 독창적
- 진보성: 단순 색상 변화를 넘어 물리적 파괴 메타포를 통한 심리적 몰입 유도

**핵심 청구항:**
1. 리뷰 데이터셋의 통계적 분산값(표준편차, 분산)을 실시간 산출하는 단계
2. 산출값이 임계치 초과 시 배경 레이어에 불연속적 선형 그래픽(균열) 생성 및 기기 진동 모듈을 제어하는 단계

---

## 2. SevenSecondCollapse (7초 붕괴)

**기술:** 실시간 평점 급락(2점↑) 감지 → 7초간 균열 효과 + 경고음 재생

**선행 기술/서비스:**
- 주식 시장 변동성 완화장치(VI) 알림 시스템, 게임 'Critical State' 연출 특허와 유사
- Steam의 리뷰 폭격 감지 시스템: 사후 그래프 표시 위주, 실시간 연출 없음

**특허 가능성: 중간**
- 기능적보다 디자인적/심미적 요소가 강함
- 기술 특허보다 **UI/UX BM 특허**로 접근 권장

**핵심 청구항:**
1. 단위 시간 내 신규 등록 평점의 가중 평균 변화율을 감시하는 단계
2. 변화율이 음의 방향 임계치(-2.0) 초과 시, 시각적 파편 효과 + 가청 신호 생성 후 타임아웃(7초) 후 일반 모드로 복귀하는 로직

---

## 3. PossessionMode (빙의 모드)

**기술:** 사용자 행동 패턴(스크롤 속도, 커서 근접) 감지 → 확률적 UI 이스터에그 (버튼 회피, 속삭임)

**선행 기술/서비스:**
- 버튼이 커서를 피하는 효과: 초기 웹 디자인의 흔한 '장난 스크립트'로 존재
- Slack 로딩 메시지, Discord 특정 클릭 패턴 시 로고 변화

**특허 가능성: 중간**
- '확률적 발생' + '사용자 특정 행동' 결합 알고리즘은 인터랙티브 UI 기술로 등록 가능

**핵심 청구항:**
1. 포인터 이동 속도, 가속도, 스크롤 빈도 기반 '긴박도' 점수 산출 단계
2. 점수에 따른 난수 생성 후 UI 요소 좌표값을 포인터 좌표 반대 방향으로 실시간 오프셋 처리하는 단계

---

## 4. RatingPrediction (평점 예측)

**기술:** 사용자 평균 평점 × 대상 평균 평점 가중 합산 → 예측 점수 + '맞추기' 도전

**선행 기술/서비스:**
- 왓챠(Watcha)의 '예상 별점', IMDb 추천 시스템 — Collaborative Filtering 기반
- 넷플릭스의 예상 별점 제공

**특허 가능성: 높음**
- 단순 예측치 제공을 넘어 '맞추기 도전(Challenge)'이라는 게임 요소를 결합하여 '데이터 라벨링(리뷰 작성)' 유도하는 BM으로서 가치 높음

**핵심 청구항:**
1. 사용자의 과거 평점 분포와 항목의 전역 평점 분포를 가중 합산하는 하이브리드 추천 엔진
2. 예측값과 실제 입력값의 오차 범위를 계산하여 보상(포인트, 배지 등)을 지급하는 보상 루프 시스템

---

## 5. ReviewFingerprint (리뷰 지문)

**기술:** 사용자 평점 분포 → 5꽃잎 레이더 차트 시각화 → 관대한/균형/엄격한 리뷰어 분류

**선행 기술/서비스:**
- TripAdvisor 리뷰어 배지(Expert 등), RPG 능력치 차트
- Letterboxd 사용자별 평점 히스토그램

**특허 가능성: 중간**
- 평점 편향성(Generosity vs. Strictness)을 '5가지 축'으로 정형화하여 시각화한 점은 UI 특허로서 독창성 있음

**핵심 청구항:**
1. 평균, 표준편차, 최빈값, 작성 빈도, 최신성 기반 5차원 벡터 산출 알고리즘
2. 산출 벡터를 다각형(Radar Chart) 형태로 렌더링하고, 특정 영역 치우침에 따라 텍스트 레이블 매핑하는 데이터 처리 방식

---

## 6. RatingStreak (연속 평가)

**기술:** 연속 평가 일수 추적 → 불꽃 아이콘 + 인텐시티 바 시각화

**선행 기술/서비스:**
- Duolingo의 'Streak', GitHub의 'Contributions', Snapchat의 'Snapstreak'
- 거의 업계 표준 패턴으로 정착

**특허 가능성: 낮음**
- '연속 일수 추적' 및 '불꽃 아이콘'은 게이미피케이션의 공공재에 가까운 표준 UI
- **디자인권으로 보호** 권장 (인텐시티 바의 시각적 변화 로직)

---

## 7. GhostReviews (유령 리뷰)

**기술:** 리뷰 0건 페이지에 스켈레톤 유령 카드 + 호버 시 CTA 전환

**선행 기술/서비스:**
- Skeleton Screen 기술: 보편화됨
- 대부분의 서비스에서 '첫 리뷰 작성' 빈 화면(Empty State) 제공

**특허 가능성: 낮음**
- 스켈레톤 UI 자체는 이미 광범위 사용으로 진보성 인정 어려움
- **창작적 UI 디자인(디자인권)** 으로 접근 권장

---

## 8. 인물 평가 모드 전환 (Adaptive UI)

**기술:** 평가 대상이 인물(정치인)일 경우 자동으로 모노크롬 톤 전환

**선행 기술/서비스:**
- YouTube 다크 모드, Spotify 앨범 아트 기반 배경색 추출(Canvas)
- 이들은 모두 사용자가 직접 선택하거나 앨범 아트 색상 추출이며, 카테고리 메타데이터 기반 선제적 분위기 전환은 없음

**특허 가능성: 높음**
- 컨텐츠 카테고리를 인식하여 시스템이 선제적으로 UI 분위기를 조정해 사용자 심리적 태도에 영향을 주는 독창적 UX

**핵심 청구항:**
1. 대상 객체의 메타데이터(Category=Politician 등)를 분석하여 UI 테마 식별자를 추출하는 단계
2. 전역 스타일시트(CSS)의 채도(Saturation) 값을 0으로 조정하고 폰트 가중치(Weight)를 증가시키는 '냉철 모드' 렌더링 엔진

## 추천 출원 전략

### 1순위: 통합 특허 (3개 기능 묶기)
> "컨텐츠의 상태 및 성격에 따라 동적으로 반응하는 적응형 리뷰 인터페이스 시스템"

묶을 기능: FaultlineFeed + SevenSecondCollapse + 인물 평가 모드 전환
- 공통 메커니즘: 데이터/메타데이터 분석 → 선제적 UI 분위기 조정 → 사용자 행동 유도

### 2순위: BM 특허
> "사용자 평가 성향 예측 기반 게임화된 리뷰 유도 시스템"

대상: RatingPrediction
- 협업 필터링 + 도전 과제 + 보상 루프의 3단계 메커니즘

### 3순위: 디자인권
- GhostReviews, RatingStreak의 시각적 표현 방식
- 출원 비용 낮고 빠른 등록 가능

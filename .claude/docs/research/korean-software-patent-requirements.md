# 한국 소프트웨어 특허 출원 요건 조사
> 작성일: 2026-04-06 | 출처: Gemini 리서치

## 1. 소프트웨어 특허의 4가지 핵심 요건

소프트웨어(SW) 특허가 성립하기 위해서는 일반적인 특허 요건 외에도 **'발명의 성립성'** 단계에서 소프트웨어와 하드웨어의 유기적 결합이 핵심적으로 요구됩니다.

1. **발명의 성립성 (Statutory Invention):** 자연법칙을 이용한 기술적 사상의 창작이어야 합니다. SW 자체는 논리적 법칙이나 수학적 알고리즘에 불과할 수 있으므로, **"소프트웨어가 하드웨어를 이용하여 구체적으로 구현"**되어야 합니다.
2. **신규성 (Novelty):** 출원 전 국내외에서 공지되었거나 공연히 실시된 발명이 아니어야 합니다.
3. **진보성 (Inventive Step):** 해당 기술 분야의 통상의 지식을 가진 사람이 기존 기술로부터 용이하게 발명할 수 없어야 합니다. 단순한 '오프라인의 온라인화'는 진보성이 부정됩니다.
4. **산업상 이용가능성 (Industrial Applicability):** 산업적으로 반복해서 제조하거나 이용할 수 있어야 합니다.

## 2. 각 요건별 구체적 판단 기준

- **성립성 판단 (핵심):** "소프트웨어에 의한 정보처리가 하드웨어를 이용하여 구체적으로 실현"되고 있는지를 봅니다. 데이터의 흐름이나 연산 과정이 하드웨어(CPU, 메모리 등) 위에서 어떻게 동작하는지가 명세서에 기술되어야 합니다.
- **진보성 판단:**
  - 단순히 종이 장부를 엑셀 형태의 데이터베이스로 옮긴 것은 진보성이 없습니다.
  - 기존 방식보다 처리 속도가 현저히 빠르거나, 데이터 저장 효율이 극대화되거나, 기존에 없던 알고리즘적 해결책이 포함되어야 합니다.

## 3. BM(비즈니스 모델) 특허의 인정 범위

- 비즈니스 아이디어 그 자체는 특허 대상이 아닙니다. 반드시 **컴퓨터 기술(시계열적인 데이터 처리 과정)**이 결합되어야 합니다.
- **판례 (대법원 2003. 5. 16. 선고 2001후3149):** "영업방법 발명이 특허를 받기 위해서는 그 방법이 컴퓨터 상에서 구현되기 위한 구체적인 수단(데이터 구조, 워크플로우 등)이 구성 요소로서 기재되어야 한다"고 판시했습니다.

## 4. UI/UX 관련 특허 사례

UI/UX는 디자인보호법(화면 디자인)과 특허법(기능적 조작 방식)으로 중복 보호가 가능합니다.

- **한국:** 삼성전자의 '밀어서 잠금해제' 기능 — 특정 궤적을 따라 터치 입력이 들어올 때 잠금이 해제되는 '기능적 프로세스'로 특허 인정
- **미국:** 아마존의 '원클릭(One-Click)' 결제 — 사용자 정보를 미리 저장해두고 단 한 번의 클릭으로 결제 프로세스를 완료하는 '단계의 축소'라는 기능적 우수성으로 특허 획득

## 5. 게이미피케이션 관련 특허 사례

- **사례:** '학습 진도에 따른 캐릭터 성장 시스템' — 학습 데이터(학습량, 정답률)가 캐릭터의 특정 스탯으로 변환되는 **알고리즘적 결합**이 있을 때 특허 가능
- **핵심:** '재미'가 아니라 '데이터 처리 방식'에 초점을 맞춰야 합니다.

## 6. 리뷰/평가 플랫폼 관련 기존 특허 현황

- 허위 리뷰 필터링 알고리즘 (리뷰 작성 패턴 분석)
- 사용자 맞춤형 리뷰 노출 (사용자의 취향 데이터와 리뷰 텍스트 매칭)
- 리뷰 작성에 따른 보상 지급(토큰 이코노미) 시스템
- 단순한 '별점 부여' 방식은 공지된 기술로 간주, 최근에는 **AI를 활용한 긍/부정 분석 및 요약 기술** 위주로 출원

## 7. 특허 출원 비용 (한국 기준)

| 항목 | 금액 |
|------|------|
| 특허청 관납료 | 약 20~50만 원 |
| 변리사 출원 수임료 | 약 150~250만 원 |
| 중간 사건 대응 | 건당 30~70만 원 |
| 등록 성사금 | 약 150~250만 원 |
| **총계** | **약 400~600만 원** |

> 스타트업/개인은 관납료 70% 감면 가능

## 8. 특허 출원 절차 및 소요 기간

1. 선행기술조사 (1~2주)
2. 명세서 작성 및 출원 (2~4주)
3. 심사 대기 (약 10~14개월 / 우선심사 신청 시 2~4개월)
4. 의견제출통지서(거절이유) 발송 — 대부분 1회 이상 발생
5. 보정서/의견서 제출 (2개월 내)
6. 등록 결정 및 공고 (총 1.5년 내외)

## 9. 주요 거절 사유

- **자연법칙 미이용 (특허법 제29조 제1항):** 인간의 경제적 규칙, 수학적 공식만 나열한 경우
- **기재 불비 (특허법 제42조):** 구현 방법이 모호하여 제3자가 해당 소프트웨어를 만들 수 없을 정도로 설명이 부족한 경우
- **진보성 결여:** "단순히 수작업을 컴퓨터로 자동화한 것에 불과하다"는 지적이 가장 많음

## 10. 최근 3년간 한국 SW 특허 심사 트렌드 (2023-2025)

1. **AI 심사 기준 강화:** AI가 학습하는 데이터의 성격과 전처리 과정이 구체적이지 않으면 거절 확률 높아짐
2. **데이터 증거주의:** 단순히 "성능이 좋아진다"가 아니라, 구체적인 알고리즘의 수치적 효율 개선 근거 요구
3. **융복합 기술 우대:** 핀테크, 헬스케어 등 타 산업과 SW가 결합된 형태의 BM 특허 출원 폭발적 증가
